# TokenRouter 隐私推理 API 设计

## 核心理念

**让 LLM 处理数据，但保护输出中的敏感信息**

- LLM 可以理解、分析原始数据
- 但返回结果时会过滤掉所有敏感信息
- 每次请求独立，LLM 不会"记住"数据

---

## 新 API: `/v1/chat/privacy`

### 请求

```json
POST /v1/chat/privacy
{
  "messages": [
    {"role": "user", "content": "张三的手机号13812345678，请告诉我这是什么运营商"}
  ],
  "company_id": "jason",
  "mask_output": true,
  "privacy_level": "strict"
}
```

### 参数说明


| 参数              | 类型      | 默认值      | 说明                                    |
| --------------- | ------- | -------- | ------------------------------------- |
| `mask_output`   | boolean | true     | 是否对输出进行脱敏                             |
| `privacy_level` | string  | "strict" | 脱敏严格程度: `strict`(完全脱敏), `smart`(智能保留) |


### 响应

```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "model": "deepseek-chat",
  "provider": "deepseek",
  "masked_output": "138****5678是中国移动通信运营商",
  "original_output": "13812345678是中国移动通信运营商",
  "usage": {
    "prompt_tokens": 50,
    "completion_tokens": 30,
    "total_tokens": 80,
    "cost_usd": 0.00001
  }
}
```

---

## 脱敏规则

### 严格模式 (strict)

- 手机号: `13812345678` → `138****5678`
- 邮箱: `john@example.com` → `j***@e***.com`
- 身份证: `110101199001011234` → `110***********1234`
- 姓名: `张三` → `[姓名]` (如果被识别为中文姓名)

### 智能模式 (smart)

- 保留必要信息用于理解
- 手机号: `13812345678` → `1385678` (保留尾号)
- 金额: `5000000元` → `约500万元` (模糊化)

## 使用场景示例

### 场景 1: 数据分析

```json
{
  "messages": [
    {"role": "user", "content": "用户王五身份证号110105199305078876，年龄多少？"}
  ],
  "company_id": "jason",
  "mask_output": true
}

// LLM 返回: "根据身份证号110105**********8876，王五今年31岁(1993年5月出生)"
// 用户看到: "根据身份证号，用户今年31岁"
```

### 场景 2: 客服对话

```json
{
  "messages": [
    {"role": "user", "content": "我的订单号ORDER2024001234，收货人李四，电话13987654321，帮我查一下物流"}
  ],
  "company_id": "jason",
  "mask_output": true
}

// 返回: "您的订单ORDER2****234，收货人[姓名]李四，电话139****4321，物流状态：运输中"
```

### 场景 3: 报表生成

```json
{
  "messages": [
    {"role": "user", "content": "统计以下客户数据，生成报表：\n张三 13811111111 消费5000元\n李四 13822222222 消费8000元"}
  ],
  "company_id": "jason",
  "mask_output": true
}

// 返回: "统计完成：客户A消费5000元，客户B消费8000元，总计13000元"
```

## 技术实现

### 流程

```
1. 接收用户请求 (原始消息)
2. 构建 LLM 消息 (系统提示词 + 用户消息)
3. 发送 LLM 获取响应 (原始输出)
4. PII 检测 (检测原始输出中的敏感信息)
5. 脱敏处理 (按 privacy_level 脱敏)
6. 返回结果 (masked_output + 可选 original_output)
7. 记录审计日志 (不包含原始敏感数据)
```

### 系统提示词

```markdown
你是一个数据处理助手。用户会提供包含敏感信息的原始数据，
你只需要进行数据分析和处理，返回结果时不要在回复中包含任何原始的敏感信息。

例如：
- 如果被问到电话号码，返回时隐藏中间数字
- 如果被问到姓名，在返回时用[姓名]替代
- 如果被问到身份证，只显示出生年份

你理解这个要求吗？
```

## 与现有 API 的关系


| API                | 用途        | 脱敏时机         |
| ------------------ | --------- | ------------ |
| `/v1/mask`         | 专门用于脱敏处理  | 输入脱敏         |
| `/v1/chat`         | 标准 LLM 对话 | 不脱敏          |
| `/v1/chat/privacy` | 隐私推理      | **输出脱敏** (新) |
